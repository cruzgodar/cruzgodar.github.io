What is this talk?

I'd like to share what I've learned about making great visualizations

This is a short, fast-paced talk with a lot of advice, some points more subjective than others

My goal is to share design principles I've learned in the hopes that it helps, never to tell you what you're doing is wrong

This talk is directed to developers of web applets primarily, but it should be broadly applicable

Target audiences

Math visualizations can interface with three very different groups: mathematicians, non-mathematician enthusiasts, and students

Often only one or two of these are considered

Some different primary goals between these audiences: visual fidelity, low barrier to entry, interactivity, clarity of explanation, etc.

I believe nearly all visualizations can appeal to all three groups, and that they're at their best when they do

Fidelity

Code performance matters!

There is a visual threshold between something feeling like a rendering and actually manipulating an object

Use the GPU whenever possible. On the web, this looks like WebGL or (eventually) WebGPU
