Section 3: Functions of Multiple Variables

After getting our bearings in 3D space and graphing objects like lines, planes, and spheres, let’s start dipping our toes into graphing proper functions of multiple variables. We already explored the notation $z = f(x, y)$ back in section 1, but we haven’t been able to turn something like $z = \sin(x) + \cos(y)$ into a graph.

Set Builder Notation

This section will see us interfacing with sets of numbers and points that are a little more complicated than ones we’ve seen in the past. In single-variable calculus, interval notation largely suffices to describe the sets of real numbers we need to deal with: for example, $[-1, 3)$ is the set of all numbers $x$ such that $-1 \leq x < 3$. We also sometimes use the union symbol $\cup$ to “add” two sets together, as in $(-\infty, -5] \cup [5, \infty)$ for the set of real numbers $x$ with $|x| \geq 5$. When we’re working with sets in $\mathbb{R}^2$ and higher, though, intervals aren’t quite enough to describe all of the sets we’re interested in. When we find ourselves describing the unit sphere as “the set of points $(x, y, z)$ in $\mathbb{R}^3$ with $x^2 + y^2 + z^2 = 1$”, we’re pushing up against the limitations of how long a sentence can be before it’s too clunky to use. To reduce some of that pressure, we’ll introduce a notation for turning a long sentence like that into a more dense but more routinely parsable sequence of symbols.

Rather than give a formal definition, we’ll introduce set builder notation by way of example. Let’s start with the unit sphere example from before: it’s given by

$$\begin{align*}\left\{ (x, y, z) \in \mathbb{R}^3 \mid x^2 + y^2 + z^2 = 1 \right\}.\end{align*}$$

Let’s take this one symbol at a time. Set builder notation uses braces, as do sets when they’re presented as lists of numbers, like $\left\{ 1, 2, 3 \right\}$ for the set containing $1$, $2$, and $3$. Within those braces, every set builder expression has a left part and a right part, separated by the vertical bar $\mid$, which we read as “such that”.

On the left, we write the general form an element from the set takes (i.e $(x, y, z)$), and where it lives. The symbol $\in$ means “element of”, and we usually just read it as “in”. All in all, $(x, y, z) \in \mathbb{R}^3$ means “all points $(x, y, z)$ in $\mathbb{R}^3$”. To the right of the separator bar, we list conditions that the points have to satisfy to be in the set. In this case, that’s the condition $x^2 + y^2 + z^2 = 1$, and so the set itself is the unit sphere in $\mathbb{R}^3$. Set builder notation is more concise than writing things in words, and it also makes us start by giving the general form of elements of the set and a familiar set in which they live, both of which help contextualize the set we’re talking about.

Functions of Two Variables

Let’s dig into functions of the form $f(x, y)$. Like functions of one variable, they have domains and ranges: the domain is the set of values that we’re allowed to plug into $f$, and the range is the set of possible outputs $z = f(x, y)$ that actually appear as an output for some input $(x, y)$.

Example: domain and range

Let $f(x, y) = \sqrt{4 - x^2 - y^2}$. What is the domain and range of $f$?

Let’s look at domain first — how could we break this function by plugging things into it? The only potential issue is the root: we need $4 - x^2 - y^2 \geq 0$, so $x^2 + y^2 \leq 4$ The domain is a set in $\mathbb{R}^2$, since the function takes in points $(x, y)$, and so we can write it in set builder notation as

$$\begin{align*}\left\{ (x, y) \in \mathbb{R}^2 \mid x^2 + y^2 \leq 4 \right\}.\end{align*}$$

That is, $f$ is defined only on the solid disk of radius $2$ centered at the origin.

What about the range? Square roots only ever produce positive numbers, so $f(x, y) \geq 0$. Also, the farther either $x$ or $y$ is away from $0$, the smaller $f$ gets, and so the largest output is $f(0, 0) = 2$. Since we can find an output of every number in between, the range is

$$\begin{align*}[0, 2] = \left\{ x \in \mathbb{R} \mid 0 \leq x \leq 2 \right\}.\end{align*}$$

On the one hand, graphing a function $z = f(x, y)$ is no more difficult than graphing $y = f(x)$: we just plug in points $(x, y)$ and plot the point $(x, y, f(x, y))$ in $\mathbb{R}^3$. On the other hand, graphing anything at all in 3D is much more challenging than graphs in $\mathbb{R}^2$, and there’s just a lot more than can be going on with a function of multiple variables. Let’s start by sketching a few graphs by hand, and then we’ll develop a tool that will help us draw and think about graphs in a more systematic manner.

Example: sketching a graph

Let $f(x, y) = \sqrt{4 - x^2 - y^2}$. Sketch a graph of $f$.

There are a number of a ways we could approach drawing this graph, but one that will consistently be useful for us is to think about its intersection with the $yz$- and $xz$-planes. That just means setting $x = 0$ and $y = 0$ in turn to produce $z = \sqrt{4 - x^2}$ and $z = \sqrt{4 - y^2}$. As a function of one variable, both have the same graph: a half-circle of radius $2$. We can therefore draw a sort of wireframe of the graph on those two planes. We can even add one more wire by finding the intersection with the $xy$-plane. That’s a little bit different: we solve $f(x, y) = 0$ to find $\sqrt{4 - x^2 - y^2} = 0$, so $x^2 + y^2 = 4$.

With this, the overall shape of the graph becomes clearer: it’s the top half of a sphere of radius $2$. Toggle it on in the Desmos graph to see! We could also see this by squaring both sides to get $x^2 + y^2 + z^2 = 4$, and remembering that squaring both sides adds potentially extraneous solutions when $z$ is negative; but since we won’t know what most graphs look like so intuitively, this is more useful to check our work.

Exercise: sketching a graph

A birding store’s monthly profit in thousands of dollars is given by $f(x, y) = 25 - (x - 1)^2 - (y - 2)^2$, where $x$ is the number of hundreds of birdfeeders sold per month, and $y$ is the number of hundreds of pairs of binoculars sold per month. Find the practical domain of $f$ (i.e. the inputs to $f$ that make sense), and sketch a graph of $f$. Where do you think $f$ is maximized?

We can’t exactly sell a negative number of birdfeeders, so a reasonable practical domain is $\left\{ (x, y) \in \mathbb{R}^2 \mid x \geq 0, y \geq 0 \right\}$. We can sketch the wireframes as before, although this time they’re a little bit more complicated. The vertical ones are

$$\begin{align*}z = 25 - (x-1)^2 - (-2)^2 = 21 - (x-1)^2,\end{align*}$$

which is a parabola opening downward with a vertex at $(1, 21)$, and the very similar graph of

$$\begin{align*}z = 25 - (-1)^2 - (y-2)^2 = 24 - (y-2)^2.\end{align*}$$

Careful not to just drop the entire $x$ and $y$ terms, but to actually set each variable to zero! The intersection with $z = 0$ is $(x - 1)^2 + (y - 2)^2 = 25$, which is a circle of radius $5$ centered at $(1, 2)$.

The result here is a little bit harder to visualize! It’s a 3D parabola-like figure whose horizontal slices are circles. The maximum is when the terms $(x-1)^2$ and $(y-2)^2$ are minimized, so at $(1, 2, 25)$.

That example and especially the exercise indicate we’d do well to develop a better way to graph and visualize functions like this! The way we intersected the graph with the horizontal plane $z = 0$ is particularly useful here, and we’ll be able to generalize and strengthen it to be much more useful.

Level Curves

The theory of functions of multiple variables has been around longer than the computers and applications we use to graph them. And as humans, we’ve dealt with them less formally a whole lot longer than we’ve had the terminology of functions. How did we go about representing 3D data in 2D forms previously — for example, the height of terrain on a map?

Well, there’s one common way that’s still very much in use today: topographical maps, or just topo maps for short. They work by drawing lines on a 2D map that indicate paths of constant height: if you walk along any line in a topo map, you’ll be staying at exactly the same height above sea level.

This map of Cone Peak in the Big Sur wilderness in California conveys the incredible steepness of the summit: every line crossed is another 100 feet of elevation gained, so the tighter the lines are bunched together, the steeper the slope there is.

Exercise: topo maps

1. Using the topo map, what do you think the area around the blue river looks like near the summit? What about the area around the river near the bottom of the map?

2. Pick any point on the map. Which direction do you need to travel so that your path is as steep as possible? How do you know?

1. The area near the top is likely much steeper, since the river is passing through more lines more quickly, and there is also not much of a valley formed by the river, since the lines don’t bend toward the river nearly at all. The opposite is true near the bottom!

2. Traveling at a right angle to the topo lines is a good bet! If traveling parallel to them keeps us at the same altitude, then traveling orthogonally should make us move uphill/downhill as quickly as possible.

Topo maps let hikers carry the data of terrain and elevation along with them in a 2D form, and we can repurpose the idea to let us more easily graph functions of multiple variables. When we found the intersection of a graph with $z = 0$, there was nothing particularly special about $0$. Repeating that process for $z = c$ lets us define a whole family of intersections:

Definition: level curve

Let $f(x, y)$ be a function of two variables. The level curve of $f$ at $z = c$ is the intersection of the graph of $z = f(x, y)$ with the plane $z = c$.

Let’s take a look at the function $z = x^2 + y^2$. To find its level curves, we just replace $z$ with $c$ to find $x^2 + y^2 = c$. That’s the equation of a circle with radius $\sqrt{c}$, and so the level curves are circles with that radius. Let’s draw the level curves for $c$ between $0$ and $10$ in the $xy$-plane to get a sense of the graph’s shape.

Here we can really see what the level curves are doing — although the closeness of the outer circles might make it seem like the height is leveling off, it’s actually indicating that that’s as steep as it gets. The 3D view lets us check that this is correct — try clickingtapping the 2D graph paper icon in the top right — but it’s valuable to be able to intuit facts about a graph from the level curves alone. For example, we can see that the graph is circular when sliced in this manner, and that it’s steepest far away from the origin.
